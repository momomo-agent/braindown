The problem

AI agents produce Markdown everywhere — project specs, meeting notes, changelogs, research, configs. You open them in VS Code and see raw syntax. You open them in a browser and lose context. You need a reader that just reads.

What BrainDown does

◇

Markdown, focused

Open a folder — only Markdown files show in the sidebar. No node_modules, no .git, no noise. Just the documents that matter.

{ }

JSON & YAML, visualized

Auto-detects structure and renders dashboards, status lists, timelines, and tables. Your config files become readable without modification.

◆

Block architecture

Every heading, paragraph, code block, and table is an independent block. Cross-block selection with Gestalt proximity grouping.

▧

Syntax highlighting

Code blocks with line numbers, language labels, and async rendering. Non-markdown files get full syntax highlighting too.

◐

Dark & Light

Follows system appearance or set manually. Every pixel considered in both modes.

⌘

Native macOS

SwiftUI + AppKit. Real NSTextView, real scroll views, real keyboard shortcuts. No Electron. No web views. No compromise.

Fits your workflow

⌘O Open folder — sidebar with file tree
⇧⌘O Open single file — no sidebar, no distraction
⌘E Toggle read / edit mode
