# DBB 用户行为场景

## S1: 首次启动
1. 打开 BrainDown
2. 看到空白界面 + 侧边栏
→ 检查：窗口大小合理吗？视觉重心在哪？有引导吗？

## S2: 打开文件夹
1. File → Open Folder，选一个包含 md 文件的目录
2. 侧边栏出现文件树
→ 检查：文件树层次清晰吗？图标区分度够吗？展开/折叠流畅吗？

## S3: 阅读短文档
1. 点击一个 50 行的 md 文件
2. 通读全文
→ 检查：标题层次、正文舒适度、代码块可读性、列表对齐

## S4: 阅读长文档
1. 打开一个 500+ 行的 md（含标题、代码块、表格、图片、引用）
2. 从头滚到尾
→ 检查：滚动流畅度、各元素间距节奏、图片加载、表格排版

## S5: 切换主题
1. View → Theme → Dark
2. 再切回 Light
→ 检查：过渡自然吗？暗色模式下代码块/引用块颜色舒服吗？有没有刺眼元素？

## S6: 调整窗口
1. 全屏 → 拖到一半宽度 → 拖到最小
→ 检查：内容始终居中吗？最小宽度下内容不溢出？侧边栏响应合理？

## S7: 切换文件
1. 在侧边栏快速点击不同 md 文件
→ 检查：切换速度、有没有闪烁、滚动位置是否重置
