# 品味标准

## 参照物

### Craft（主要参照）
- 安静到消失的排版，工具感极低
- 标题用衬线体，正文无衬线，形成优雅对比
- 块元素跟正文过渡自然，没有"突然插了一个框"的感觉
- 大量留白，从容不迫

### Bear（辅助参照）
- 代码块圆角精致，背景色极淡
- 列表圆点小而精致
- 侧边栏简洁，hover 反馈克制

### Notion（辅助参照）
- 标题衬线体的书卷气
- 信息密度适中，功能清晰
- 块元素边界感明确但不粗暴

## 设计原则

1. **接近性节奏** — 标题前留白 ≥ 标题后留白的 3 倍
2. **层次靠弱化** — 次要信息用灰色弱化，主要信息不需要特别强调
3. **间距系统** — 全局只用 4 / 8 / 12 / 16 / 24 / 32
4. **颜色克制** — 正文不用纯黑（light: 0.15, dark: 0.85），标题比正文略深
5. **字体对比** — H1/H2 衬线体（Georgia），H3 以下和正文用系统无衬线
6. **行高舒适** — 正文行高 1.7-1.8 倍，代码块 1.3-1.4 倍

## 代码块品味标准（重点）

### 容器
- 独立圆角容器，圆角 8-10pt
- 背景色极淡（light: ~0.96, dark: ~0.12），不抢正文注意力
- 微妙的 1px 边框（light: 0.90, dark: 0.18）增加容器感
- 内边距四边一致 16pt

### 顶栏
- 语言标签在左上角，小字 11pt，柔和灰色
- 复制按钮在右上角，hover 才显示，点击后短暂显示 ✓
- 顶栏和代码内容之间有细分隔线或微妙间距

### 代码内容
- 等宽字体 13.5pt（SF Mono 或 Menlo）
- 行高 1.35-1.4
- Highlightr 语法高亮，主题跟随 light/dark
- 行号在左侧，灰色，右对齐，跟代码之间有间距
- 行号不可选中

### 整体感觉
- 代码块像一张"卡片"嵌在文档流里，有明确的边界但不突兀
- 跟 Notion 的代码块放一起，质感要接近

## 表格品味标准（重点）

### 容器
- 独立圆角容器，圆角 8pt
- 微妙的 1px 外边框
- 内部用极细分隔线（0.5pt）

### 表头
- 背景色区分（light: ~0.96, dark: ~0.16）
- 字重 semibold
- 底部有稍粗的分隔线（1pt）跟数据行区分

### 数据行
- 交替行色（极微妙的差异，不是斑马纹）
- hover 行高亮（柔和的蓝色/灰色）
- 单元格内边距 12pt 水平，8pt 垂直

### 列
- 列宽根据内容自适应
- 最小列宽保证可读性
- 文字左对齐，数字右对齐

### 整体感觉
- 表格像一个精致的数据展示组件，不是"文字被格子框住了"
- 跟 Notion 的表格放一起，质感要接近

## 验收清单

### 排版
- [ ] 标题前留白 ≥ 后留白 × 3
- [ ] 所有间距值都在 {4,8,12,16,24,32} 中
- [ ] 正文颜色不是纯黑/纯白

### 代码块
- [ ] 圆角容器 + 微妙边框
- [ ] 语言标签 + 复制按钮
- [ ] 行号显示
- [ ] 语法高亮正确
- [ ] 四边 padding 视觉一致
- [ ] 不抢正文注意力

### 表格
- [ ] 圆角容器 + 微妙边框
- [ ] 表头背景色 + semibold
- [ ] hover 行高亮
- [ ] 列宽自适应
- [ ] 单元格内边距舒适

### 阅读体验
- [ ] 连续阅读 500 行 md 不觉得累
- [ ] 标题层次一眼可辨
- [ ] 暗色模式无刺眼元素
